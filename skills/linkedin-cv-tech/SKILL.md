## LP.1 — Comment LinkedIn fonctionne (l'algorithme, le SSI)

### LinkedIn comme moteur de recherche

LinkedIn fonctionne comme **Google pour les profils pros**. Les recruteurs tapent des mots-clés, un algo rankent les profils. Les champs les plus indexés (poids décroissant) :

```
1. Titre du profil (headline)       ← poids maximal
2. Compétences / Skills
3. Poste actuel (titre exact)
4. About / Infos
5. Descriptions d'expériences
6. Education
7. Certifications
```

### Le SSI (Social Selling Index)

LinkedIn attribue un score 0-100 sur 4 dimensions :
- **Établir votre marque professionnelle** (profil complet, contenu publié)
- **Trouver les bonnes personnes** (recherches, visites de profils)
- **Échanger des informations** (engagement sur les posts)
- **Construire des relations** (connexions, recommandations)

> Score > 70 = profil "All-Star" = visible beaucoup plus souvent dans les recherches recruteuses.

### Ce que vérifie un recruteur en 7 secondes

```
Regard 1 : Photo + Titre
Regard 2 : Employeur actuel + Durée
Regard 3 : 2 premières lignes de l'About
Regard 4 : Compétences principales (top 3)
```

Si ces 4 éléments ne convainquent pas → profil fermé.

## LP.3 — Titre du profil / Headline — 220 caractères

**Le champ le plus indexé de LinkedIn. Le plus vu. Le moins bien utilisé.**

### Formule recommandée

```
[Rôle actuel] | [Stack / Domaine clés] | [Contexte chiffré] | [Tagline personnelle]
```

### Règles d'or

1. **Ne jamais laisser le défaut** (LinkedIn met juste "Poste chez Entreprise")
2. **Utiliser les 220 caractères** — espace précieux et indexé
3. **Séparer par |** — lisibilité + parsing algorithmique
4. **Inclure les mots-clés que les recruteurs tapent** — titres de postes, technologies, domaines
5. **Une tagline humaine** — ce qui vous rend mémorable
6. **Ne pas** : "Passionné par...", "Cherche nouvelles opportunités", "Guru", "Ninja", "Rockstar"

### Formule par profil tech/IA

#### CTO / VP Engineering / Deputy CTO

```
CTO | Data, IA & Cloud | Scale-up B2B [N pers.] | Ex-[Titre] | [Tagline]
Deputy CTO & Gatekeeper | Data & IA | Snowflake · LLM · Agentic | Groupe [N] pers.
```

#### Data Engineer / Architect

```
Data Engineer | Snowflake · dbt · Airbyte · Python | Data Lakehouse & Pipelines ELT
Senior Data Architect | Modern Data Stack · Snowflake · Databricks | Gouvernance & Performance
```

#### AI/ML Engineer

```
ML Engineer | LLM & GenAI en Production | RAG · Vector Stores · Python · AWS SageMaker
AI Engineer | Agentic AI · Prompt Engineering · n8n | Automatiser l'intelligence métier
```

#### No-code / Low-code / Ops

```
No-Code Builder | n8n · Make · Airtable · Notion | Workflows & Automatisation pour PME
RevOps / BizOps | HubSpot · Zapier · Power Platform | Aligner Sales, Marketing et Data
```

#### Product Owner / TPM (tech)

```
Technical Product Manager | IA & Data Products · LLM · APIs | B2B SaaS | Livrer vite, bien, souvent
PO Data & IA | Snowflake · Cortex · Power BI | Traduire les besoins métier en produits data
```

#### Exemple complet (profil de référence)

```
Deputy CTO & Gatekeeper | Data, IA & Snowflake | Ex-Directeur Technique (20 pers.) | Groupe B2B 4000 pers.
Proactif par nature, efficient par design, augmenté par l'IA
```
*(216/220 — quasiment parfait)*

### Anti-patterns headline tech

| À éviter | Pourquoi | Remplacer par |
|----------|---------|--------------|
| "Passionné par les données" | Non indexé, non différenciant | "Data Engineer | Snowflake · dbt · Python" |
| "Expert en IA" | Trop vague | "AI Engineer | LLM · RAG · Python · AWS" |
| "En recherche active" | Gaspille les chars | Utiliser Open to Work à la place |
| "Guru / Ninja / Wizard" | Unprofessionnel | Intitulé de poste + stack |
| "Directeur Informatique" seul | Pas searchable | "CTO / VP Engineering | Cloud · Data · IA" |

## LP.4 — About / Infos — 2 600 caractères

**Seules les 3-4 premières lignes sont visibles** sans cliquer "voir plus". Le hook est critique.

### Structure en 7 blocs (adaptée tech/IA)

```
[BLOC 1 — HOOK] 1-2 phrases max. Formule choc ou contradiction.
Ne doit pas commencer par "Je m'appelle..." ou "Je suis passionné..."

[BLOC 2 — IDENTITÉ PROFESSIONNELLE] Ce que vous faites concrètement.
Rôle, périmètre, équipe, enjeux. Chiffres si possible.

[BLOC 3 — PREUVES / RÉALISATIONS] 3-5 bullet points.
Chaque bullet = 1 réalisation concrète, idéalement chiffrée.

[BLOC 4 — STACK TECHNIQUE] Vos outils, plateformes, langages.
Organisé par domaine : Data / IA / No-code / Cloud

[BLOC 5 — APPROCHE / PHILOSOPHIE] Ce qui vous différencie dans la manière de travailler.
Court, personnel, mémorable.

[BLOC 6 — CE QU'ON VOUS AMÈNE] Vos domaines d'intervention préférés.
Ce que les gens viennent chercher chez vous.

[BLOC 7 — CTA / CONTACT] Comment vous joindre, pourquoi vous connecter.
```

### Exemple de hook efficace (style éprouvé en 2026)

```
❌ "Je suis un ingénieur data passionné par les nouvelles technologies..."
✅ "Je vois des solutions là où d'autres voient des réunions."
✅ "La data, c'est mon terrain de jeu."
✅ "L'IA, c'est mon accélérateur."
✅ "J'ai découvert Snowflake chez [X]. J'ai compris que j'allais passer à une autre échelle."
```

### Longueur recommandée

```
About minimal viable : 500-800 mots (suffisant pour l'algo)
About optimal : 1 500-2 000 mots  
Attention : 2 600 chars ≈ 400-450 mots. Ne pas bourrer — 70% rempli, fluidement = mieux que 100% étouffant.
```

### Keywords à intégrer naturellement dans l'About

Pour un profil data/IA, inclure au moins une occurrence de :
- Votre titre de poste cible
- 3-5 technologies clés de votre stack
- 2-3 domaines (Data Engineering, IA Générative, Automatisation...)
- 1-2 soft skills démontrés (pas juste nommés)

### Ce que l'About n'est PAS

```
❌ Un copier-coller de votre CV
❌ Une liste de postes ("J'ai travaillé à...")
❌ Un paragraphe RH générique ("Mes valeurs sont l'intégrité et l'excellence")
❌ De la modestie excessive ("Je suis encore en apprentissage...")
```

## LP.5 — Descriptions d'expérience — 2 000 caractères par poste

### Formule STAR adaptée à la tech

```
[CONTEXTE] Qui, où, quelle équipe, quel périmètre.
[MISSION] Ce dont vous étiez responsable (pas les tâches — la responsabilité).
[RÉALISATIONS] 3-5 bullets. Format = Verbe d'action + quoi + résultat chiffré.
[STACK] Technologies et outils utilisés dans ce rôle.
```

### Verbes d'action tech recommandés

| Domaine | Verbes |
|---------|--------|
| **Architecture / Build** | Architecturé, Conçu, Développé, Déployé, Industrialisé, Migré, Refactorisé |
| **Data** | Modélisé, Ingéré, Transformé, Optimisé, Normalisé, Orchestré, Catalogué |
| **IA** | Implémenté, Fine-tuné, Prompté, Évalué, Déployé en production, Intégré |
| **Management** | Piloté, Fédéré, Accompagné, Formé, Recadré, Recruté, Organisé |
| **Transverse** | Simplifié, Prouvé, Industrialisé, Uniformisé, Aligné, Priorisé, Arbitré |
| **Résultat** | Réduit de X%, Augmenté de X%, Accéléré de X jours, Économisé Xk€ |

### Template de description d'expérience

```
**Périmètre :** [N] personnes · [N] BU · Budget [X]€ · Contexte [startup/scale-up/grand groupe]

**Mission :** [1-2 phrases sur la responsabilité globale]

**Réalisations clés :**
▶ [Verbe] [quoi] → [résultat chiffré]
▶ [Verbe] [quoi] → [résultat chiffré]
▶ [Verbe] [quoi] → [résultat ou impact qualitatif notable]
▶ [Verbe] [quoi] → [résultat chiffré]

**Stack :** [Techno 1] · [Techno 2] · [Techno 3] · [Outil 1] · [Outil 2]
```

### Exemple (profil data/IA senior)

```
**Périmètre :** 4 personnes · Gatekeeper transverse pour 2 Comex (1k+ pers.) · Groupe B2B 4 000 pers.

**Mission :** Simplifier les choix technologiques IA & Data, prouver rapidement ce qui marche et industrialiser ce qui tient.

**Réalisations clés :**
▶ Déployé 3 chatbots métier (externe + interne) → 0 à 200 utilisateurs actifs en 6 semaines
▶ Industrialisé le pipeline ELT Snowflake + Airbyte → -70% de temps de préparation data pour les BU
▶ Formé 50 collaborateurs aux usages IA → taux d'adoption +40% vs objectif
▶ Construit le Design System IA du groupe → socle commun pour 12 équipes produit

**Stack :** Snowflake · n8n · Airbyte · PowerBI · Cortex · RAG · Python · Claude · OpenAI
```

## LP.6 — Compétences / Skills : stratégie tech/IA

### Les 2 niveaux de compétences LinkedIn

```
Niveau 1 — Compétences affichées dans le résumé : 5 max, visibles sans cliquer
→ Choisir les 5 les plus cherchées par les recruteurs pour votre cible

Niveau 2 — Compétences totales : jusqu'à 100
→ Être exhaustif ici pour maximiser la surface de recherche
```

### Les 5 skills "résumé" pour chaque profil type

| Profil | 5 compétences résumé recommandées |
|--------|----------------------------------|
| **CTO / Deputy CTO** | Stratégie IT · Architecture technique · Data Engineering · IA Générative · Management tech |
| **Data Engineer** | Snowflake · dbt · Python · SQL · Data Pipeline |
| **AI Engineer** | LLM · RAG · Python · Prompt Engineering · MLOps |
| **Data Analyst** | SQL · Power BI · Python · Analyse de données · Storytelling data |
| **No-code/Ops** | n8n · Make/Integromat · Airtable · Power Automate · Automatisation |
| **Product Owner tech** | Gestion de produit · Agile/Scrum · APIs · SQL · UX |

### Règles compétences

```
□ Associer chaque compétence à au moins 1 expérience (champ "compétence de ce poste")
□ Passer les évaluations LinkedIn disponibles (badge top 30% = signal fort)
□ Demander des endorsements sur vos top 5 (pas tous — les 5 qui comptent)
□ Utiliser les termes anglais OU français selon votre cible (pas les deux en doublon)
```

## LP.8 — Soft skills : le bon vocabulaire tech

**Ne jamais** écrire "je suis dynamique", "passionné" ou "team player" — sans preuve.  
**Toujours** montrer le soft skill par une réalisation ou un comportement observable.

### Tableau soft skills → formulation professionnelle

| Soft skill réel | Formulation creuse ❌ | Formulation démontrée ✅ |
|-----------------|---------------------|------------------------|
| Vulgarisation | "Je suis pédagogue" | "J'ai formé 50 non-techniciens à utiliser les outils IA" |
| Leadership | "Je suis un leader" | "J'ai constitué et managé une équipe de 20 personnes dont 5 leads" |
| Pragmatisme | "Je suis orienté résultats" | "Je POC et prouve en jours au lieu de laisser diverger des mois" |
| Vision | "J'ai une vision stratégique" | "J'ai transformé une mission temporaire en function critique pour le groupe" |
| Adaptabilité | "Je m'adapte facilement" | "CDI → CDD → Directeur en 18 mois dans 3 contextes différents" |
| Communication | "Excellent communicant" | "Référent IA pour 2 Comex — je traduis la complexité en décisions" |
| Curiosité | "Passionné par l'innovation" | "J'explore chaque nouvel LLM en sortie — Claude, Kimi, Grok, DS, Gemini" |

### Soft skills à fort impact dans un profil tech senior

```
Simplification du complexe — "je simplifie là où les autres jargonnent"
Pensée systémique — vision end-to-end au-delà des silos
Orientation produit vs projet — construire ce qui dure
Frugalité d'implémentation — partir du besoin, pas de la techno
Biais pour l'action — POC avant PowerPoint
Autonomisation des équipes — l'objectif est de se rendre dispensable
Clarté décisionnelle — arbitrer vite avec des données incomplètes
```

## LP.9 — Featured (À la une) + sections complémentaires

### Featured / À la une

C'est la vitrine visible juste sous l'About. Premier élément que les recruteurs cliquent.

**Ce qu'y mettre (par ordre d'impact) :**

```
1. Un article ou post avec > 500 reactions — preuve d'expertise et d'audience
2. Une vidéo de démo d'un projet / talk en conférence
3. Un projet GitHub ou portfolio avec présentation
4. Un article de blog / Medium sur votre domaine
5. Une publication LinkedIn Native (article long format)
6. Un lien vers votre site perso / page de services
```

**Ce qu'il ne faut PAS y mettre :** articles d'autres personnes, posts sans engagement, photos de soirée d'équipe.

### Certifications : construire son "badge wall" tech

Certifications les plus valorisées par profil :

| Profil | Certifications prioritaires |
|--------|---------------------------|
| **Cloud / Infra** | AWS Solutions Architect · Google Cloud Professional · Azure Solutions Architect · Terraform Associate |
| **Data** | Snowflake SnowPro Core/Advanced · Databricks Data Engineer · dbt Analytics Engineer |
| **IA/ML** | Google Machine Learning Engineer · AWS ML Specialty · DeepLearning.AI / Coursera (Andrew Ng) |
| **Agile/PM** | PMP · PSPO · CSM · SAFe · ICAgile |
| **Sécurité** | AWS Security · CISSP · CEH |
| **No-code** | HubSpot Certifications · Salesforce Certifications · Power Platform Developer |

### Recommandations : comment en obtenir

```
Protocole en 3 étapes :
1. Écrire une recommandation sincère à quelqu'un → déclenche souvent la réciproque
2. Envoyer un message personnalisé : "Je prépare mon profil pour [contexte] — tu pourrais en 2-3 lignes témoigner de [aspect spécifique] ?"
3. Proposer un draft (optionnel) — ça facilite et guide sans forcer

Cibles prioritaires :
→ Managers N+1 (crédibilité hiérarchique)
→ Clients ou partenaires (crédibilité orientée résultats)
→ Collaborateurs directs (crédibilité humaine et managériale)
```

## LP.10 — CV tech : structure et format

### Format universel

```
□ 1 page si < 7 ans d'expérience
□ 2 pages max si > 7 ans (jamais 3)
□ PDF uniquement (pas .docx — les ATS l'acceptent, et le rendu est stable)
□ Marges 17-20mm, police sans serif (Calibri, Inter, Helvetica, Source Sans)
□ Taille police : 10-11 corps, 13-14 titres
□ Pas de photo (CV tech international) OU photo professionnelle (CV FR)
□ URL LinkedIn, GitHub, portfolio en header
```

### Structure recommandée

```
[HEADER]
Prénom NOM — Titre du poste cible
[Ville] · [Email] · [Téléphone] · linkedin.com/in/prenom-nom · github.com/...

[RÉSUMÉ EXÉCUTIF — 3-4 lignes]
Ce que vous faites, pour qui, avec quel impact. Mots-clés denses.

[EXPÉRIENCES — ordre anti-chronologique]
Poste | Entreprise | Dates | Lieu
→ Bullet 1 : réalisation chiffrée
→ Bullet 2 : réalisation chiffrée
→ Bullet 3 : impact / contexte
Stack : [technologies]

[FORMATION]
Titre | École | Année

[COMPÉTENCES TECHNIQUES]
Data : Snowflake · dbt · Python · SQL · Power BI...
IA : LLM · RAG · Prompt Engineering · n8n...
Cloud : AWS · Azure · Docker · Terraform...

[CERTIFICATIONS] (si pertinentes)

[LANGUES]
Français : natif | Anglais : courant (TOEIC [score])
```

### ATS (Applicant Tracking System) : les règles

**49% des CV ne passent pas les ATS** faute de mauvais formatage.

```
✅ Texte pur (pas de tableaux, pas de colonnes multi-zones pour les sections principales)
✅ Titres de sections standards ("Expérience professionnelle", "Formation", "Compétences")
✅ Dates en format cohérent (MM/AAAA ou Jan. 2024)
✅ Intitulés de postes correspondant aux recherches recruteurs
✅ Répéter les mots-clés de l'offre dans la description du poste visé
❌ Pas de header/footer pour les infos importantes (non lu par ATS)
❌ Pas d'images, icons, infographies dans les zones de texte
❌ Pas de colonnes CSS (certains ATS lisent de gauche à droite en ignorant les colonnes)
```

## LP.11 — CV tech : rédiger les bullet points de postes

### La formule XYZ (Google)

```
[Verbe d'action] [X — ce que j'ai fait] pour [Y — pour qui / dans quel contexte] en atteignant [Z — résultat chiffré]
```

### La formule CAR (Challenge · Action · Résultat)

```
Challenge : Quel était le problème ?
Action : Qu'est-ce que j'ai fait concrètement ?
Résultat : Quel impact mesurable ?
→ Condensé en 1-2 lignes
```

### Exemples de bullets CV tech

```
✅ Migré 15 pipelines ETL Oracle vers Snowflake + Airbyte → réduction du temps de traitement de 65%
✅ Déployé un assistant IA interne (RAG + Claude) → adoption 200 utilisateurs en 8 semaines, NPS +40 pts
✅ Constitué et managé une équipe data de 8 personnes (4 Data Engineers + 2 Analysts + 2 ML Engineers)
✅ Industrialisé un design system IA → socle commun pour 12 équipes, réduction TTM de 3 semaines
✅ Conçu l'architecture lakehouse (Snowflake + dbt + Airflow) pour 4 BU → +30% de fiabilité des rapports direction
```

### Chiffres à mobiliser (même si on ne les a pas "officiellement")

```
→ Taille d'équipe managée
→ Taille de l'entreprise / périmètre (lits, clients, CA, pers.)
→ Volume de données traitées (Go/To par jour, millions de lignes)
→ Réduction de délai (de X jours à Y jours)
→ Réduction coûts (X% ou Xk€)
→ Adoption (N utilisateurs, N% taux d'utilisation)
→ Nombre de projets livrés / temps
→ Budget géré
```

## LP.12 — Anti-patterns LinkedIn & CV

### Anti-patterns LinkedIn

| Anti-pattern | Impact | Solution |
|-------------|--------|---------|
| **Titre = "Directeur à [Entreprise]"** | Non searchable | Titre = rôle + domaine + stack |
| **About vide ou < 300 chars** | Profil non-All-Star, non indexé | Minimum 500-800 chars |
| **Pas de photo ou photo informelle** | -14x de visites | Photo pro en 400x400 |
| **Compétences sans endorsements** | Crédibilité nulle | Demander 5+ endorsements sur le top 5 |
| **Pas de certifications** | Manque de signaux récents | Au moins 2-3 certifs récentes |
| **Expériences sans descriptions** | L'algo ne peut pas indexer | Mini 3 bullets par poste |
| **Pas de Featured** | Opportunité manquée | Même 1 post fort → grande visibilité |
| **Activité = 0 post depuis 2 ans** | Profil "mort" dans l'algo | 1 post/semaine minimum |
| **Skills en doublon EN/FR** | Dilue les endorsements | Choisir une langue par skill |

### Anti-patterns CV tech

| Anti-pattern | Impact | Solution |
|-------------|--------|---------|
| **Objectif professionnel en header** | Inutile, prend de la place | Résumé exécutif à la place |
| **"Maîtrise de MS Office"** | Anachronique en 2026 | Enlever sauf si requis |
| **Liste de 30 technologies** | Noise, incrédible | Top 15 pertinents uniquement |
| **Bullets en phrases longues** | Non scannable | 1 bullet = max 2 lignes, verbe d'action |
| **Dates floues "2020-2023"** | Gaps invisibles ou ambigus | "Jan. 2020 – Fév. 2023" |
| **CV en .docx ou avec colonnes** | ATS fail | PDF, mise en page linéaire |
| **Compétences sans niveau** | Pas actionnable | "Python (confirmé)" ou par catégorie |
| **Pas de liens (GitHub, LinkedIn)** | Pas de preuve | Header avec liens actifs |
| **Mêmes bullets pour tous les postes visés** | Non personnalisé | 1 version par type de poste cible |

## Checklist profil LinkedIn All-Star (à valider avant publication)

```
Identité
□ Photo professionnelle (400x400, fond neutre)
□ Photo de bannière personnalisée
□ URL personnalisée (linkedin.com/in/prenom-nom)
□ Titre du profil : 150-220 chars, mots-clés + tagline

Contenu
□ About : 500 mots minimum, hook fort, stack technique, CTA
□ Poste actuel : description > 500 chars, bullets chiffrés, stack
□ 3+ postes précédents remplis avec descriptions
□ Formation complète (toutes les sections)
□ 5+ compétences avec endorsements sur les top 5

Crédibilité
□ 3+ recommandations publiées
□ 2+ certifications récentes (< 3 ans)
□ Featured : au moins 1 contenu fort
□ Évaluations de compétences passées (top 30%)

Activité
□ SSI > 65
□ Dernière publication < 30 jours
```

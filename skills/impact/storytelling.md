# Impact — Storytelling & Communication

## Quand activer ce skill

- Préparer une présentation, un pitch, un discours ou une conférence
- Rédiger un texte qui doit marquer les esprits
- Rendre un message technique plus humain et mémorable
- Concevoir les slides et le scénario d'une intervention

# I. Storytelling & Communication

> Les histoires sont le mode natif de l'esprit humain.

## I.1 Commander's Intent

**Source** : US Army Field Manual ; Stanley McChrystal, *Team of Teams* (2015)

**Citation** : *"Describe not just your plan, but what you want to achieve and why."*

**Principe** :
Dans l'armée américaine, chaque ordre inclut le **commander's intent** : l'objectif final et pourquoi il importe. Quand les plans s'effondrent (et ils s'effondrent toujours), les soldats peuvent improviser en gardant l'intention en vue.

**Structure** :
1. **Situation** : Contexte et pourquoi c'est important
2. **Mission** : Ce qu'on doit accomplir
3. **Commander's Intent** : L'état final désiré et pourquoi

**Application** :
- Commencer toute présentation par l'intention finale, pas l'agenda
- "À l'issue de cette session, je veux que vous partiez avec X"
- Donner l'intention permet à l'audience d'orienter son attention

## I.2 Sparkline (Nancy Duarte)

**Source** : Nancy Duarte, *Resonate* (2010)

**Principe** :
La structure narrative la plus persuasive alterne entre **ce qui est** (réalité actuelle) et **ce qui pourrait être** (monde idéal), créant une tension narrative qui appelle l'action.

**Structure Sparkline** :

```
Ce qui       Ce qui       Ce qui
pourrait     pourrait     est
être         être         
    \       /    \       /
Ce         Ce         Ce
qui        qui        qui
est        est        est
    
APPEL À L'ACTION
```

**Exemples iconiques** : MLK *"I Have a Dream"* ; Steve Jobs iPhone keynote

**Application** :
- Alterner entre la réalité inconfortable d'aujourd'hui et la vision séduisante de demain
- Terminer par une invitation à traverser le pont vers "ce qui pourrait être"

## I.3 Feuilleton & Cliffhanger

**Source** : Alexandre Dumas — structure feuilleton ; Adam Silberstein — tension narrative

**Principe** :
La structure feuilleton (sériel avec cliffhangers) maintient l'engagement sur la durée en multipliant les boucles ouvertes (voir D.6 Effet Zeigarnik).

**Structure du cliffhanger** :
1. Moment de tension maximal
2. Interruption délibérée
3. Promesse de résolution

**Application** :
- Terminer chaque session ou chapitre sur "la prochaine fois vous découvrirez X qui change tout"
- Introduire un problème en début de séquence que vous ne résolvrez qu'à la fin
- Les titres "Question à 1 million" créent le cliffhanger avant même de commencer

## I.4 Les 3 Plots Narratifs

**Source** : Chip Heath & Dan Heath, *Made to Stick* (2007) — section récits ; Annette Simmons, *The Story Factor* (2001)

**3 archétypes narratifs applicables en formation et communication professionnelle** :

| Plot | Structure | Effet psychologique |
|------|-----------|-------------------|
| **Challenge** | Adversité → effort → dépassement | Inspiration, courage |
| **Connection** | Deux personnes (ou idées) se rejoignent malgré les obstacles | Appartenance, empathie |
| **Créativité** | Problème insoluble → insight inattendu → solution nouvelle | Curiosité, espoir |

**Application** :
- Choisir le plot en fonction de l'émotion cible
- Plot Challenge pour mobiliser face à un changement difficile
- Plot Créativité pour inspirer face à un problème "impossible"

## I.5 Picture Superiority Effect

**Source** : Allan Paivio — Dual Coding Theory (1971) ; Lionel Standing (1973)

**Principe** :
Les images sont mémorisées environ 6 fois mieux que les mots seuls. Combinées à du texte, la mémorisation est maximale.

**Chiffres** :
- Standing (1973) : 2 500 images présentées → reconnaissance à 90% après 3 jours
- Information seule : 10% mémorisée après 3 jours
- Information + image : 65% mémorisée après 3 jours

**Application** :
- Une image forte > une liste de 10 bullets
- Si abstract → trouver la métaphore visuelle correspondante
- Les schémas > les listes
- Éviter les images décoratives : chaque image doit porter l'information

## I.6 Règle 10-20-30

**Source** : Guy Kawasaki, *The Art of the Start* (2004)

**Principe** :
La présentation optimale :
- **10 slides** maximum
- **20 minutes** maximum
- **30 points** minimum (taille de police)

**Logique** : Le cerveau ne peut retenir qu'une fraction de ce qu'on lui presente. Moins = plus de rétention. 30 points = le présentateur ne peut plus coller son nez sur ses slides.

**Application** :
- Pour une présentation d'1h : max 20 slides
- Taille de police minimale : 24pt (pas 30 si contexte différent)
- Se forcer à réduire = clarifier l'essentiel

## I.7 Billboard Test

**Source** : Reynolds, *Presentation Zen* (2008) ; Duarte, *Slide:ology* (2008)

**Principe** :
Chaque slide doit pouvoir être lue et comprise en **3 secondes** par quelqu'un qui passe en voiture. Une idée = un slide.

**Critères** :
- Une seule idée maîtresse par slide
- Titre = l'idée principale (pas juste un sujet)
- Visuel dominant > texte dominant
- Pas plus de 6 mots dans un titre de slide

**Application** :
- Test : couvrir le contenu de la slide et lire seulement le titre → le message passe-t-il ?
- Si non → réécrire le titre en phrase messagère

## I.8 Le Hook (5 Types)

**Source** : Doug Stevenson, *Story Theater Method* (2008) ; Nancy Duarte ; diverses sources storytelling professionnel

**Principe** :
Les premières 30 secondes d'une présentation déterminent si l'audience "reste avec vous" ou décroche. Le hook = dispositif d'ouverture qui capture l'attention immédiatement.

**5 types de hooks** :

| Type | Description | Exemple |
|------|-------------|---------|
| **Question rhétorique** | Poser une question à laquelle tout le monde a une réponse privée | "Combien d'entre vous ont passé plus de 2h sur leur email hier ?" |
| **Chiffre choc** | Statistique contre-intuitive ou surprenante | "80% des projets IA échouent à l'adoption" |
| **Histoire personnelle** | Anecdote courte avec tension et résolution | "Il y a 3 ans, j'ai failli perdre mon poste à cause d'un email mal formulé" |
| **Promesse concrète** | Ce que l'audience aura dans X minutes | "Dans 45 minutes, vous aurez un prompt que vous utiliserez dès demain" |
| **Contradiction paradoxale** | Deux vérités en tension | "L'IA va vous faire gagner du temps... et vous en coûter beaucoup plus si vous l'utilisez mal" |

## I.9 Boucle Ouverte (Nested Loops)

**Source** : Storytelling avancé ; technique d'hypnose narrative (Milton Erickson) ; J.J. Abrams — Mystery Box

**Principe** :
Commencer plusieurs histoires/questions sans les terminer immédiatement pour maintenir plusieurs boucles d'attention ouvertes. Le cerveau reste engagé jusqu'à la résolution de chaque boucle.

**Structure** :
```
Ouvrir histoire A
  Ouvrir histoire B
    Ouvrir histoire C
      [contenu principal]
    Fermer histoire C
  Fermer histoire B
Fermer histoire A
```

**Application** :
- Commencer par une histoire personnelle, passer au concept, revenir finir l'histoire
- "Avant de vous dire la fin de cette anecdote, laissez-moi vous montrer X"
- Maximum 3 boucles imbriquées sans confusion

## I.10 Hero's Journey (Campbell + Formation)

**Source** : Joseph Campbell, *The Hero with a Thousand Faces* (1949) ; Christopher Vogler, *The Writer's Journey* (1992)

**12 étapes de Campbell** (version adaptée formation) :

| Étape | Monde ordinaire | Transformation |
|-------|----------------|----------------|
| **Monde ordinaire** | Le monde AVANT | "Voici comment vous travaillez aujourd'hui..." |
| **Appel à l'aventure** | Le défi apparaît | La question, le problème |
| **Refus de l'appel** | Résistance | Normal — les 9 visages de la peur |
| **Rencontre avec le mentor** | Guide, outil, méthode | Vous, les outils, les pairs |
| **Franchissement du seuil** | Engagement | Le premier pas concret |
| **Alliés et épreuves** | Pratique | Les exercices |
| **Caverne profonde** | L'épreuve principale | Le défi le plus difficile |
| **Ordalie** | Transformation | Le moment "ça y est, je comprends" |
| **Récompense** | La compétence acquise | La démonstration de maîtrise |
| **Chemin du retour** | Retour vers le quotidien | Application en contexte réel |
| **Résurrection** | Le test final | Comment vont-ils utiliser ça ? |
| **Retour avec l'élixir** | Transmission | "Partagez ça avec votre équipe" |

**Application** :
- L'apprenant est le Héros, pas le formateur
- Le formateur = le Mentor (Gandalf, pas Frodo)
- Concevoir chaque module comme un voyage en 12 étapes

## I.11 StoryBrand (7 éléments)

**Source** : Donald Miller, *Building a StoryBrand* (2017)

**Principe** :
La marque/le formateur n'est pas le héros — le client/l'apprenant est le héros. La marque est le guide.

**7 éléments** :

| Élément | Question |
|---------|----------|
| 1. **Personnage** | Qui est le héros ? (client / apprenant) |
| 2. **Problème** | Quel problème externe, interne, philosophique ? |
| 3. **Guide** | Vous, l'outil, la méthode |
| 4. **Plan** | Quelles étapes concrètes ? |
| 5. **Appel à l'action** | Que doit faire le héros ? |
| 6. **Éviter l'échec** | Qu'est-ce qui se passe si on ne change pas ? |
| 7. **Succès** | À quoi ressemble la vie après ? |

**Application** :
- Toute communication de changement peut être structurée en 7 éléments
- Mettre le participant/client EN PREMIER dans la narration
- "Voici ce que VOUS allez pouvoir faire" > "Voici ce que notre solution fait"

## I.12 Règles Pixar (4 applicables en formation)

**Source** : Emma Coats (ex-Pixar storyteller) — 22 Rules of Storytelling ; adaptées Chip Heath

**4 règles Pixar applicables** :

| Règle | Formulation | Application |
|-------|-------------|-------------|
| **Rule #4** | "Once upon a time... every day... until one day... because of that... until finally... ever since" | La structure de tout case study pédagogique |
| **Rule #7** | Finissez avec la conclusion, commencez avec la tension | Ouvrir sur le problème, pas sur la solution |
| **Rule #10** | Séparer votre déception de votre valeur | Les erreurs dans l'histoire ne définissent pas le personnage |
| **Rule #17** | Pas d'accidents dans les bonnes histoires | Chaque élément du réel qu'on choisit doit servir le sens |

**Structure Rule #4 appliquée** :
```
Il était une fois [situation initiale]...
Chaque jour [routine/normalité]...
Jusqu'au jour où [perturbation/déclencheur]...
À cause de ça [première conséquence]...
À cause de ça [escalade]...
Jusqu'à ce que finalement [résolution]...
Depuis lors [nouvel état]
```

## I.13 Presentation Zen

**Source** : Garr Reynolds, *Presentation Zen* (2008, 2019)

**Principes clés** :

**Avant la conception** :
- Travailler sur papier ou tableau blanc d'abord (pas sur PowerPoint)
- Identifier L'idée principale — une seule phrase

**Design** :
- Espace blanc = signal, pas vide
- Contraste fort (texte / arrière-plan)
- Images authentiques > clip-art
- Alignement rigoureux

**Présentation** :
- Connecter avec l'audience AVANT de regarder les slides
- Les slides sont un support, pas un script
- Toucher, pause, regarder (avant de parler)

**6 principes** :
1. Restraint (retenue dans la conception)
2. Simplicity (une idée par slide)
3. Naturalness (connexion humaine)
4. Elegance (pas d'ornementation inutile)
5. Subtlety (suggestion > démonstration)
6. Impermanence (les slides passent, l'impact reste)

## I.14 Slide:ology

**Source** : Nancy Duarte, *Slide:ology: The Art and Science of Creating Great Presentations* (2008)

**Principes de design visuel** :

**Hiérarchie visuelle** :
- Grand = important
- En haut = premier
- Contraste = différence

**Loi de proximité** (Gestalt) : Ce qui est proche = lié

**Règle des tiers** : Placer l'élément principal sur une intersection des tiers

**Types de slides et leur usage** :
| Type | Usage |
|------|-------|
| **Titre** | Ancre le thème, met le ton |
| **Transition** | Signal de changement de section |
| **Bullet** | Liste d'éléments parallèles (max 4-5) |
| **Diagramme** | Relations, processus, structures |
| **Data** | Chiffres — un point par graphique |
| **Photo** | Émotion, contexte humain |
| **Quote** | Autorité, mémorisation |
| **Blank** | Pause, retour au présentateur |

**Les 5 questions à chaque slide** :
1. Quelle est l'idée maîtresse ?
2. Est-ce que visual et texte se complètent ou se répètent ?
3. Y a-t-il du désordre inutile ?
4. L'audience peut-elle le lire à 5 mètres ?
5. Ce slide serait-il plus efficace en le divisant en deux ?
